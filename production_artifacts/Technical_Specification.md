# 概要 (Executive Summary)
AIやITエンジニア向けの最新情報を効率的に収集・閲覧できるRSSリーダーアプリケーションです。主要なテック系メディアや個人の技術ブログからRSSフィードを取得し、モダンで直感的なUI（ダークモード対応、グラスモーフィズムなどのリッチなデザイン）を通じて、快適な情報収集体験を提供します。

# 要件 (Requirements)
## 機能要件
- **RSSフィードの取得・解析**: 定期的に指定されたRSS/Atomフィードから記事のタイトル、リンク、日付などを取得。
- **記事の一覧表示**: 取得した新着記事を時系列またはカテゴリ別に表示。
- **検索とフィルタリング**: キーワードやタグ（AI、Web、インフラなど）による記事の絞り込み。
- **ブックマーク（お気に入り）機能**: 気になる記事を保存して後から読めるようにする。
- **フィードソースの管理**: 新しいRSSフィードURLの追加や既存フィードの削除。

## 非機能要件
- **パフォーマンス**: 記事一覧の高速なレンダリングとページネーション（または無限スクロール）。
- **レスポンシブデザイン**: PC、タブレット、スマートフォンで最適化されたレイアウト。
- **モダンなUI/UX**: ダークモードを基調とし、マイクロアニメーションを取り入れたプレミアムなデザイン。Tailwind CSSは使用せず、洗練されたVanilla CSSでスタイリングする。

# アーキテクチャと技術スタック (Architecture & Tech Stack)
- **フルスタックフレームワーク**: **Next.js (App Router)**
  - 理由: サーバーサイドレンダリング(SSR)やAPIルートを1つのプロジェクトで統合でき、高速なページロードが可能なため。
- **データベース**: **SQLite** (ORMとして **Prisma**)
  - 理由: 初期開発のスピードとローカル環境での扱いやすさを重視。将来的なPostgreSQLへの移行も容易。
- **RSSパーサー**: **rss-parser** (Node.js向けライブラリ)
  - 理由: 多数のRSS/Atomフォーマットに柔軟に対応でき、実績があるため。
- **スタイリング**: **Vanilla CSS (CSS Modules)**
  - 理由: 独自の洗練されたデザインシステムを構築し、ダイナミックなUIを実現するため。

# 状態管理 (State Management)
- **サーバー状態**: Next.jsのServer Componentsおよび `fetch` キャッシュを利用し、データベースからの記事取得を最適化。
- **クライアント状態**: Reactの `useState` と `Context API`（必要に応じて `Zustand`）を使用。ブックマークのトグル操作などのUI状態を即座に反映（オプティミスティック更新）させるために利用します。
